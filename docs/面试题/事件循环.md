# 事件循环

1. `Javascript` 引擎的单线程特性
同一时间内只能执行一个任务, 所有代码都在主线程的调用栈中按顺序执行

2. 事件循环(`Event Loop`)的核心作用
协调同步代码执行, 调度异步任务(宏任务和微任务)以及浏览器的渲染时机

3. 调用栈(`Call Stack`)的工作机制
后进先出, 同步任务直接入栈执行, 执行完毕后出栈. 当栈清空时, 事件循环才会处理异步队列

4. 常见的宏任务(`Macrotask`)
- `setTimeout`
- `setInterval`
- `I/O` 操作
- `UI 交互事件`

5. 常见的微任务(`Microtask`)
- `Promise.then`
- `Promise.catch`
- `Promise.finally`
- `MutationObserver`
- `process.nextTick` (node)
- `await` 后的代码

6. 宏任务和微任务的执行顺序
- 在每一个宏任务执行完毕后, 需要清空所有的微任务后(接着渲染完毕)才会执行下一个宏任务

7. 浏览器渲染时机
- 渲染发生在清空微任务之后或者触发显示器刷新周期

8. `script` 标签在事件循环中的本质
- 每一个 `script` 标签都是一个宏任务

9. `async` 表达式中的 `await` 后的代码执行时机
- 立即执行其同步代码, 后续代码被包装成一个微任务 `Promise.then` 放入微任务队列

10. 微任务执行期间产生新的微任务怎么处理
- 持续执行至清空所有微任务才会进入下一个宏任务

11. 同步长任务对 `UI` 渲染的影响
- 同步长任务会阻塞 `UI` 渲染和页面的交互事件, 造成页面卡死 `Jank`

12. `setTimeout(fn, 0)` 的执行
- 将 `fn` 放入下一个或更晚的宏任务队列

13. `Promise` 的执行
- 同步执行, 回调函数 `then` `catch` `finally` 放入微任务异步执行

14. `requestAnimationFrame` 的执行
- 在微任务之后渲染之前执行

15. `requestIdleCallback` 的执行
- 在事件循环空闲(无任务且当前帧有剩余时间)时执行的低优先级的任务

16. `nodejs` 的事件循环和浏览器有什么区别
- `nodejs` 有 `process.nextTick` 优先于 `Promise.then`
- `nodejs` 没有 `requestAnimationFrame` 和 `requestIdleCallback`
- `nodejs` 的宏任务队列分为 `timers` `pending callbacks` `idle, prepare` `poll` `check` `close callbacks`

17. 如果在循环中递归调用微任务会怎么样
- 会导致微任务队列无限增长, 阻塞后续宏任务的执行页面卡死

18. 一轮事件循环中宏任务和微任务的最大执行数量
- 宏任务: 1个
- 微任务: 无限制

19. `VSync` 信号对事件循环的影响
- `VSync` 信号是浏览器渲染的刷新周期, 一般为 `16.67ms`
- `VSync` 信号会阻塞事件循环, 导致事件循环的执行时间变长

20. 事件循环的休眠状态
- 当所有任务队列都清空时浏览器会进入休眠状态, 直到下一个任务的到来

21. `UI` 渲染的最佳时机
- 在微任务之后显示器刷新 `VSync` 信号产生的时机

22. 真实点击事件和调用 `.click()` 的区别
- 真实点击事件会视为一个独立的宏任务
- 代码调用则是调用栈同步执行

23. `queueMicrotask` 的优势
- 比 `Promis.then` 创建的微任务更直接减少了 `Promise` 创建的开销

24. 微任务抛出错误后对后续宏任务的影响
- 不会影响后续宏任务的执行, 每个任务之间有强隔离性

25. 为什么 `js` 依旧坚持单线程
- 历史原因, 早期 `js` 的设计目标就是
- 避免多线程带来的复杂性, 如竞态条件和死锁
- 简化异步编程模型
