# 基础
1. `[] == ![]` 结果？
右侧的 `![]` 由于数组是对象则一定是 `truthy` 得到右侧为 `false`
一侧为对象一侧为基本类型，左侧正常走 `OrdinaryToPrimitive` 转换原始值得到 `''`
一侧为字符串一侧为布尔值，都转数字，结果变成 `0 == 0`
结果是 `true`

2. `{} + []` 结果？
首先 `{}` 是个空代码块直接忽略
`+[]` 将 `[]` 转原始值得到 `''`
`+''` 将 `''` 转数字拿到 `0` 
`+0` 
结果为 `0`

3. `[] + {}` 结果？
这里的 `{}` 就不是代码块了，而是视为一个字符串拼接表达式
`[]` 转原始值拿到 `''`
结果为 `'[object object]'`

4. `[1] - 1` 和 `[1] + 1` 结果？
`[1] - 1` 是减法数字操作，会将前面的数组转原始值后去转数字
`[1] + 1` 前面的原始值拿到 `'1'` 就变成字符串拼接了
结果为 `0` 和 `'++'`

5. `[1] == 0` 和 ``[0] == 0`` 结果？
一侧数字一侧字符串，字符串转数字
`[1] == 0` 对象转原始值得到 `'1' == 0` 再转数字得到 `1 == 0`
`[0] == 0` 对象转原始值得到 `'0' == 0` 再转数字得到 `0 == 0`
结果为 `false` 和 `true`

6. `null == 0` 和 `null == undefined ` 结果？
规范中 `null` 和自什么以及 `undefined` 外，其他不论什么类型 `==` 比较都是返回 `flase`
结果为 `false` 和 `true`

7. `null == 0` 和 `null >= 0` 结果？
`null` 除了和 `null` `undefined` 是相等的，和其他比较都是 `false`
`>=` 比较会先把 `null` 转为数字 `0`，`0 >= 0`
结果为 `false` 和 `true`

8. `undefined` 在大小比较和相等比较中有没有转成 `NaN`
大小比较中会转`NaN`，相等比较中不会

9. `'2' > '12'` 结果？
字符串之间比较会按下标顺序比较编码顺序，而不是直接使用长度比较或者转数字后比较
结果为 `true`
